How to JavaScript
JavaScript ist eine objektorientierte Programmiersprache für Clientside-Programming. Das bedeutet, dass das Programm auf den Computer des Nutzers (Clients) geladen wird und erst dort ausgeführt. Für Serverside-Programming können Python, PHP, Java aber auch JavaScript benutzt werden. JS wird vorallem für die Interaktion mit einem HTML-Dokument benutzt.

Erste Schritte
JavaScript ist sehr vielseitig, doch der Hauptverwendungszweck ist, eine Website interaktiv zu machen. Dafür gibt es die DOM-API. Eine API ist ein Paket, das dir Funktionen für einen bestimmten Zweck zur verfügung stellt. Die DOM ist automatisch in jeder Website in Betrieb.
Wo schreibe ich mein Programm?
Du kannst dein Program direkt in das HTML schreiben. Die andere Möglichkeit ist, das Programm in eine extra File zu schreiben. Man benutzt für beide Arten den gleichen Tag, nur mit anderen Attributen:
<script></script> <script src="filename.js"></script>
In das erste Beispiel schreibst du direkt dein Programm rein, sei es alert("Hello World") oder andere Dinge. In das 2. Beispiel wird üblicherweise nichts reingeschrieben, denn alles, das da drin steht, wird angezeigt, wenn der Browser des Clients die Ausführung von JS deaktiviert hat. Es ist empfohlen, seine script-tags in den HEAD zu schreiben, nicht aber in den BODY. Technisch ist beides möglich.
Reihenfolge beachten!!!
Ein HTML-Dokument wird von oben nach unten gelesen, so auch der JS-Code. Will man aber HTML-Elemente ansprechen, die weiter unten im Dokument stehen, werden sie nicht gefunden.

Um das zu umgehen, sorgt man entweder dafür, dass das Script-Element am Ende des Dokuments im BODY steht, oder, dass die Elemente erst nach dem Laden der kompletten Seite abgefragt werden. Letzteres erfolgt über das onload-Event.
Wartung und Tests
Die Browser Microsoft Edge und Google Chrome, aber auch Mozilla Firefox beinhalten eine Entwicklungsumgebung für HTML, JS und CSS. Sie besteht vor allem aus einer HTML-Elemente-Ansicht mit den entsprechend als Text assoziierten Styles und einer JavaScript-Console. Letztere ist praktisch um nachzuschauen, wie bzw. ob das Programm funktioniert hat. Ausserdem werden dort die Syntaxfehler und andere Fehler, die im Programm sind, angezeigt. Du erreichst sie, indem du die Tastenkombination Strg + Shift + i benutzst.
