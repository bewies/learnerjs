Funktionen
Funktionen sind Code-Blöcke, die ausgeführt werden, wenn man sie ruft. Sie haben den Zweck, lange und unübersichtliche Codes zu verübersichtlichen und den selben Code mit verschiedenen Eingangswerten wiederbenutzbar zu machen.

Syntax
Es gibt 2 Arten eine Funktion zu deklarieren: Entweder als Variable oder mit dem function-Schlüsselwort. Die zweite Methode ist gängiger, dennoch ist es gut zu wissen, was eine Funktion genau ist, und, was sie kann.
Das function-Schlüsselwort
Um einer Funktion zu deklarieren, brauchst du folgenden Code:
function functionName(param1, param2, ...){ // some Code // some more Code // eventuell ein return-Statement }
Diese Funktion kannst du jetzt rufen, indem du einfach den Funktionsname mit passenden Parametern in deinen Code oder in die Console schreibst. Du kannst diese Funktion auch einem Event zuweisen, doch dazu später.
functionName("Hello", "World");
Parameter
Was eine Funktion so praktisch macht, ist, dass sie für verschiedene Inputs verschiedene Outputs haben können. Diese Parameter werden direkt in die runden Klammern geschrieben. In der Definition der Funktion werden an ihrer Stelle Variablen benutzt.
writeToElement-Beispiel
Nehmen wir an, wir wollen eine Funktion, die 2 Inputs, also Parameter, hat:
Ein String, der bestimmtwo reingeschrieben werden soll
und
Ein Element, in das der String reingeschrieben wird.

Überlegen wir uns die Befehlskette:
Gegeben ist el und str.
Setze das innerHTML von el auf str.

Mit dem im Hinterkopf, können wir nun unseren Code schreiben. Doch zuallererst brauchen wir ein Markup:
<html> <head> <title>My First useless Function</title> </head> <body> <div id="demo1"> Ich bin 1 </div> <div id="demo2"> Ich bin 2 </div> <div id="demo3"> Ich bin 3 </div> <div id="demo4"> Ich bin 4 </div> <!-- Funktion wird hier definiert. --> <script> ... </script> </body> </html>
Wir wollen nun also eine Funktion schreiben. Um sie gleich testen zu können, wenden wir die Funktion direkt unter der Definition auf ein Element an.
let el2 = document.querySelector("#demo2"); function writeToElement(el, str){ el.innerHTML = str; } writeToElement(el2, "Ich bin durch eine Funktion manipuliert");
Nützliche Funktionen
Das Prinzip einer Funktion sollte mittlerweile klar sein. Doch lass uns doch mal etwas Nützliches damit machen. Würde es uns nicht extrem helfen, wenn wir nicht immer document.queryS... schreiben müssten, sondern einfach get? Dazu könnten wir einfach eine Funktion schreiben, die uns das Element der Suche zurückgibt. Dazu benötigen wir das return-Statement.
Das return-Statement
Das return-Statement ist zum Beenden von Funktionen da. Beim return-Statement stoppt eine Funktion und gibt einen Wert zurück. Dieser Wert wird wie "eingesetzt" an den Ort, wo die Funktion steht, die gerufen wurde. Am besten lässt sich das an der Funktion document.querySelector() sehen.
let erstesDiv = document.querySelector("div");
Die Variable erstesDiv bekommt keineswegs den Wert document.querySelector(...), sondern den Wert, der für die Funktion eingesetzt wird. Die Funktion document.querySelector(...) hat nämlich am Ende seiner Definition ein return-Statement, das das gefundene Element zurückgibt.
Libraries
Libraries sind JS-files, die dir eine Reihe voll Funktionen bieten, die du dann in deinem Code benutzen kannst. Sie sind meist einfacher zu lesen oder haben mehr Anwendungsmöglichkweiten. Um eine Library zu benutzen, musst du sie einfach nur im HEAD verlinken.
<script src="https://code.example.com/libraries/myLibrary.js"></script>
Eigene Library
Wir sind nun in der Lage, eine Funktion zu schreiben, sie zu rufen, ihr andere Inputs zu geben und zuguterletzt einen Wert von ihr zurückzubekommen. Lass uns dieses Wissen nun anwenden und unsere kleine Mini-Library schreiben. Sie soll die Funktionen get, getAll und getBy beinhalten.
Verlinkung
Um unsere Library zu verlinken, musst du ersteinmal die Datei anlegen. Im Beispiel wird die Library selector.js heissen. Speichere die Datei also an einem bestimmten Ort und verlinke sie in deinem HTML mit einem passenden Link.
<html> <head> <title>My first self-made Library</title> <!-- Library wird hier verlinkt. --> <script src="path/to/library/selector.js"></script> </head> <body> <div id="demo1"> Ich bin 1 </div> <div id="demo2"> Ich bin 2 </div> <div id="demo3"> Ich bin 3 </div> <div id="demo4"> Ich bin 4 </div> <!-- Programm wird hier ausgeführt. --> <script> ... </script> </body> </html>
Jetzt haben wir die Library erfolgreich verlinkt, nur steht noch nichts darin. Um das zu ändern, widmen wir uns jetzt der Funktionen in der Library.
selector.js
Fangen wir mit der get-Funktion an. Wir haben einen Input namens i und einen Output: Das Element, das bei der Funktion document.querySelector(i) herauskommt. Das ganze im Code geschrieben sieht folgendermassen aus:
function get(sel){ return document.querySelector(sel); }
Für die Funktion getAll sieht der Code ganz ähnlich aus. Wir müssen nur zwei kleine Dinge anpassen:
1. Den Funktionsnamen.
2. Die querySelector-Funktion zu querySelectorAll ändern.
function getAll(sel){ return document.querySelectorAll(sel); }
Der Code für die Funktion getBy ist ein bisschen komplexer, aber völlig machbar. Doch lass ihn uns erst in Worten überlegen. Wir haben einen Input type und einen sel. Ersteres steht für den Typ, welches getElement... Funktion ausgeführt werden soll. Letzteres für den Selektions-String, der in den Selektor eingesetzt wird.

Wenn type ist "id",
gib getElementById(sel) zurück,
sonst, wenn type ist...

Das geht so weiter, bis wir alle Typen durch sind. Es werden zwei mögliche Umsetzungen hier zur Verfügung stehen, das Erste wird ein (if, else if, else)-Statement benutzen, während das andere den Switch-Block benutzen wird.

Hier die erste Umsetzung:
function getBy(type, sel){ if( type == "id" ){ return document.getElementById(sel); }else if( type == "class" ){ return document.getElementsByClassName(sel); }else if( type == "tag" ){ return document.getElementsByTagName(sel); }else{ console.error("Type kann nur die Werte id, class und tag annehmen."); } }
Und hier die zweite:
function getBy(type, sel){ switch(type){ case "id": return document.getElementById(sel); break; case "class": return document.getElementsByClassName(sel); break; case "tag": return document.getElementsByTagName(sel); break; default: console.error("Type kann nur die Werte id, class und tag annehmen."); } }
Die erste Variante ist übersichtlicher und leichter und schneller zu bauen, weswegen diese Weise von vielen bevorzugt wird.
Switch-Block
Ein Switch-Block ist ein gängiges Mittel in einigen Programmiersprachen, der einem erlaubt, für verschiedene Zustände einer Variable einen anderen Codeblock auszuführen. Das break-Statement ist nötig, damit die anderen Fälle nicht mehr überprüft werden. Es bedeutet: "Breche hier ab!".
